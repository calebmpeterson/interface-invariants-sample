IMPORTANT: Always state when you read or apply a `ux-invariant`

## Accessibility

- All interactive elements must be reachable and operable via keyboard: `./references/accessibility-invariant-001.md`

## Destructive Actions

- Destructive Actions Require Explicit Confirmation: `./references/destructive-actions-invariant-001.md`

## Feedback

- Direct the user's attention to the first invalid input when they attempt to submit data: `./references/feedback-invariant-001.md`

## Input

- Use appropriate input type for the data expected from the user: `./references/input-invariant-001.md`

## Navigation

- Use clear/consistent iconography and labels: `./references/navigation-invariant-001.md`
